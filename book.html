A Beginner's Guide to Propositional Logic

Imagine a type of math where you don’t use numbers like 2, 5, or 100. Instead, you only have two values: True (represented by 1) and False (represented by 0). Boolean Algebra and Propositional Logic are simply the rules for doing math with these two values.

We use this logic to take basic facts, combine them, and figure out if the overall conclusion is true or false.

1. The Building Blocks: Atomic Propositions

Before we do any logic, we need things to apply logic to. These are called Atomic Propositions.

An atomic proposition is a basic statement that can be either True or False.

  • Example: "Cats are mammals." (True)
  • Example: "Cars fly." (False)

To make things easier, instead of writing out full sentences, we use variables like $x$, $y$, or $z$ to represent them. At this stage, we don’t worry about whether $x$ is actually true or false; we just know it has the potential to be either.

2. The Logical Operators (The "Math" Symbols)

In regular math, we have plus, minus, and multiply. In Boolean logic, our primary tools are three operators: AND, OR, and NOT.

  • NOT ($\neg$): This is a "flipper." It flips the truth value of whatever follows it.
    • If $x$ is True (1), then $\neg x$ is False (0).
    • If $x$ is False (0), then $\neg x$ is True (1).
  • AND ($\land$): For an AND statement to be true, both parts must be true.
    • $1 \land 1 = 1$ (True AND True = True)
    • Any other combination (like True AND False) results in 0 (False).
  • OR ($\lor$): For an OR statement to be true, at least one part must be true.
    • $1 \lor 0 = 1$ (True OR False = True)
    • Important: This is an "inclusive OR". If both parts are true, the result is still true. It only results in 0 if both parts are false.

3. Writing Formulas (Syntax)

Just like you can combine numbers and math symbols into complex equations (like $2 + (3 \times 4)$), you can combine atomic propositions and logical operators into formulas.

Example formula: $\neg x \land (y \lor z)$

Order of Operations: Just like multiplication comes before addition in regular math, logic has a priority order:

  1. NOT ($\neg$) is calculated first.
  2. AND ($\land$) is calculated second.
  3. OR ($\lor$) is calculated last.

(You can always use parentheses to change this order, just like in regular math).

4. Finding the Truth (Semantics and Valuations)

A formula by itself doesn't have a truth value until you assign True (1) or False (0) to its variables. The act of assigning these values is called a valuation.

Let's look at an example. Suppose our formula is: $\neg x \land y$
Now, let's do a valuation where $x = 1$ (True) and $y = 1$ (True).

  1. First, plug the numbers in: $\neg 1 \land 1$
  2. Apply the NOT rule: $\neg 1$ becomes $0$. Now we have $0 \land 1$.
  3. Apply the AND rule: $0 \land 1$ is $0$.

Result: Under this valuation, the formula is False (0).

5. The Three Types of Formulas

If you test a formula against every possible combination of True and False for its variables, the formula will fall into one of three categories:

  1. Tautology: A formula that is always True, no matter what.
    Example: $x \lor \neg x$ ("It is raining OR it is not raining." That statement is always true!).
  2. Contradiction: A formula that is always False, no matter what.
    Example: $x \land \neg x$ ("It is raining AND it is not raining." Impossible!).
  3. Satisfiable (Non-tautological): A formula that is sometimes True and sometimes False, depending on the variables. Most formulas fall here.

6. The "Primary Colors" of Logic

You might be wondering: what about other logical ideas, like "Exclusive OR" (XOR - meaning one or the other, but not both) or "If/Then" statements?

The beautiful thing about Boolean logic is that AND, OR, and NOT are all you need. They are like the primary colors of logic. You can build any logical rule or truth table just by combining these three.

For instance, two formulas are considered equivalent if they yield the exact same results. You can use this to express complex concepts using only our basic operators:

  • XOR can be perfectly written as: $(x \lor y) \land \neg(x \land y)$
    (Translation: "One or the other is true, AND they aren't both true.")
  • De Morgan’s Laws show how NOT interacts with AND/OR:
    • $\neg(x \land y)$ is the exact same as $\neg x \lor \neg y$
    • $\neg(x \lor y)$ is the exact same as $\neg x \land \neg y$

Turning a Truth Table into a Formula

Because AND, OR, and NOT can build anything, you can invent a completely random truth table, and easily write a formula for it. How?

  1. Look at the truth table and find only the rows where the final output is True (1).
  2. For each of those rows, write an AND statement linking the exact conditions needed to get that row (e.g., if $x$ is 0 and $y$ is 1, you write $\neg x \land y$).
  3. Link the statements for all the "True" rows together using ORs.

Summary
Propositional Logic is just a highly structured way to determine truth. By breaking complex statements into variables ($x, y$), connecting them with simple rules (AND, OR, NOT), and systematically plugging in 1s and 0s, we can logically map out every possible scenario to see exactly when a statement is true and when it is false.

Propositional Knowledge Representation

Here is a comprehensive and easy-to-understand breakdown of Propositional Knowledge Representation.

Think of this topic as teaching a computer how to be a detective: how to take a set of clues, figure out if they make sense together, and logically deduce new facts from them.

1. The Basics: Situations and Clues

In logic, a formula (like "It is raining" or "The road is wet") doesn't have a built-in truth value. It is only true or false depending on the situation (which logicians call a valuation).

  • Think of a valuation as one specific parallel universe or "scenario."
  • A piece of knowledge is a formula we know is true.

The Knowledge Base (KB)

A Knowledge Base is simply a collection of formulas (clues) that we know are true. Our goal is to use the KB to rule out impossible scenarios. If a scenario (valuation) makes every single formula in the KB true, we say that scenario satisfies the KB.

Consistency: Does the story make sense?

The very first thing we must ask about a Knowledge Base is: Is this even possible?

If a KB contains contradictory clues (e.g., "It is raining" AND "It is not raining"), no scenario can satisfy it.

  • If there is at least one scenario where all the formulas in the KB are true, the KB is consistent.
  • To check consistency, we could draw a giant truth table of every possible scenario. But if we have 100 variables, the computer would take over $10^{18}$ years to check them all! We need a smarter, faster method.

2. The Tableaux Algorithm: The Detective's Strategy

To solve this speed problem, logicians use the Tableaux Algorithm. Instead of generating every possible universe, this algorithm breaks complex formulas down into their simplest parts to see if a valid scenario can be built.

Setting the Stage (NNF)

Before starting, we translate our formulas into Negation Normal Form (NNF). This just means we push all the "NOTs" (negations) as deep into the formula as possible, so they only attach to basic variables.

  • Example: "NOT (A or B)" becomes "NOT A and NOT B".
  • The simplest pieces (like $x$ or $\neg x$) are called literals.

The Decomposition Rules

The algorithm uses two simple rules to break down formulas:

  1. The AND Rule ($\land$): If your KB says "A AND B", both must be true. The algorithm simply breaks this into two separate facts: "A is true", "B is true", and adds them both to your list of clues.
  2. The OR Rule ($\lor$): If your KB says "A OR B", we have a fork in the road. The algorithm creates branches—two alternate timelines. In one branch, we assume A is true. In the other, we assume B is true.

You keep applying these rules until every complex formula is broken down into simple literals. Once no more rules can be applied, the tableau is considered saturated.

3. Reading the Results: Open vs. Closed

Once the algorithm has broken everything down, we look at the branches (the alternate timelines) to see what we are left with.

  • Closed Branches: If a branch contains an obvious contradiction (for example, it requires $x$ to be true AND $\neg x$ to be false at the same time), that timeline is impossible. We call this branch closed.
  • Open Branches: If a branch contains no contradictions, it represents a perfectly valid scenario that satisfies all our clues. We call this branch open.

The Final Verdict:

  • If all branches close, the KB is entirely impossible. It is inconsistent.
  • If at least one branch is open, the KB makes sense. It is consistent, and the open branch actually tells you how to make it true!

4. Guarantees of the Algorithm

When relying on this algorithm, logicians need guarantees that it works perfectly. The Tableaux Algorithm is:

  • Sound: It doesn't give false positives. If it says a branch is open and consistent, it truly is.
  • Complete: It doesn't give false negatives. If a valid scenario exists, the algorithm will find it.
  • Terminating: It will never get stuck in an infinite loop. Because each step removes an "AND" or an "OR", the formulas get simpler every time, guaranteeing it will eventually finish.

5. Reasoning: Moving from "Possible" to "Guaranteed"

Checking if a story makes sense (consistency) is great, but usually, we want to extract new, guaranteed facts from our knowledge. This is called Entailment.

  • Possibility just asks: Could this formula be true given what we know?
  • Entailment asks: Is this formula GUARANTEED to be true given what we know?

If our Knowledge Base entails a formula (written as $KB \models \phi$), it means that in every single scenario where the KB is true, this formula is also true.

The Magic Trick for Entailment (Theorem 6)

How do we test for entailment without checking every universe? We use a brilliant trick: Proof by Contradiction.

If we want to prove that our KB guarantees a fact ($\phi$), we:

  1. Assume the exact opposite of the fact ($\neg \phi$).
  2. Toss that assumption into our Knowledge Base.
  3. Run the Tableaux algorithm.

If the algorithm says "This is impossible!" (inconsistent), it means the opposite of our fact completely breaks the universe. Therefore, our original fact must be absolutely true.

In short: $KB$ entails $\phi$ if and only if $KB \cup \{\neg \phi\}$ is inconsistent.

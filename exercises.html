Solution

We can explicitly guarantee the use of at least three propositional variables (let's use \(x, y,\) and \(z\)) like so:

\((x \land y \land z) \lor \neg(x \land y \land z)\)

Question:

Build a formula over four propositional variables which is true if and only if any two of the input variables is true.

Solution

Let the four propositional variables be \(a, b, c,\) and \(d\). The phrasing "if any two [...] is true" typically distinguishes between two subtle meanings in logic. Here is the formula based on the most direct interpretations:

You just require finding any two un-negated variables combined by an AND.

Build a formula that represents the Boolean function expressed by the following table:

\(x\) \(y\) \(z\) \(\varphi\)
0001
0010
0101
0111
1000
1011
1100
1111

Solution

  1. Identify the minterms (rows where \(\varphi = 1\)):
    • Row 1: \(x=0, y=0, z=0 \implies (\neg x \land \neg y \land \neg z)\)
    • Row 3: \(x=0, y=1, z=0 \implies (\neg x \land y \land \neg z)\)
    • Row 4: \(x=0, y=1, z=1 \implies (\neg x \land y \land z)\)
    • Row 6: \(x=1, y=0, z=1 \implies (x \land \neg y \land z)\)
    • Row 8: \(x=1, y=1, z=1 \implies (x \land y \land z)\)
  2. Combine to form the Disjunctive Normal Form (DNF):
    \(\varphi = (\neg x \land \neg y \land \neg z) \lor (\neg x \land y \land \neg z) \lor (\neg x \land y \land z) \lor (x \land \neg y \land z) \lor (x \land y \land z)\)
  3. Simplify the formula (grouping terms):
    • Combine rows 1 and 3: \((\neg x \land \neg y \land \neg z) \lor (\neg x \land y \land \neg z) \equiv \neg x \land \neg z\)
    • Combine rows 4 and 8: \((\neg x \land y \land z) \lor (x \land y \land z) \equiv y \land z\)
    • Combine rows 6 and 8: \((x \land \neg y \land z) \lor (x \land y \land z) \equiv x \land z\)

    Putting it together yields the simplified formula:

    \(\varphi = (\neg x \land \neg z) \lor (y \land z) \lor (x \land z)\)

    (Note: This can be further factored to \(\varphi = (\neg x \land \neg z) \lor (z \land (x \lor y))\) )

Exercise 1.7. Formula Properties (**)

Let \(\varphi\) and \(\psi\) be two propositional formulas. Check which of the following statements are true, providing a justification:

  1. if \(\varphi\) and \(\psi\) are satisfiable, then \(\varphi \land \psi\) is satisfiable;
  2. if \(\varphi\) and \(\psi\) are satisfiable, then \(\varphi \lor \psi\) is satisfiable;
  3. if \(\varphi\) and \(\psi\) are satisfiable, then \(\varphi \land \neg\psi\) is satisfiable;
  4. if \(\varphi\) and \(\psi\) are tautologies, then \(\varphi \land \psi\) is a tautology;
  5. if \(\varphi\) and \(\psi\) are contradictions, then \(\varphi \lor \psi\) is a contradiction;
  6. if \(\varphi\) is a tautology, then \(\varphi \to \psi\) is satisfiable;
  7. if \(\varphi\) is a tautology, then \(\psi \to \varphi\) is a tautology.

Solution

  1. if \(\varphi\) and \(\psi\) are satisfiable, then \(\varphi \land \psi\) is satisfiable;
    False. Justification: Counterexample: Let \(\varphi = p\) and \(\psi = \neg p\). Both are independently satisfiable, but \(\varphi \land \psi \equiv p \land \neg p\), which is a contradiction (never satisfiable).

  2. if \(\varphi\) and \(\psi\) are satisfiable, then \(\varphi \lor \psi\) is satisfiable;
    True. Justification: If \(\varphi\) is satisfiable, there is at least one truth assignment that makes it True. Under this same assignment, the disjunction \(\varphi \lor \psi\) will evaluate to True, making it satisfiable.

  3. if \(\varphi\) and \(\psi\) are satisfiable, then \(\varphi \land \neg\psi\) is satisfiable;
    False. Justification: Counterexample: Let \(\varphi = p\) and \(\psi = p\). Both are satisfiable, but \(\varphi \land \neg\psi \equiv p \land \neg p\), which is a contradiction.

  4. if \(\varphi\) and \(\psi\) are tautologies, then \(\varphi \land \psi\) is a tautology;
    True. Justification: A tautology is True under all truth assignments. Since \(\varphi\) and \(\psi\) are both always True, \(\varphi \land \psi\) will evaluate to \(\text{True} \land \text{True} \equiv \text{True}\) under all assignments.

  5. if \(\varphi\) and \(\psi\) are contradictions, then \(\varphi \lor \psi\) is a contradiction;
    True. Justification: A contradiction is False under all assignments. Because \(\varphi\) and \(\psi\) are always False, \(\varphi \lor \psi\) evaluates to \(\text{False} \lor \text{False} \equiv \text{False}\) under all assignments.

  6. if \(\varphi\) is a tautology, then \(\varphi \to \psi\) is satisfiable;
    False. Justification: Counterexample: Let \(\psi\) be a contradiction (e.g., \(q \land \neg q\)). Because \(\varphi\) is a tautology (always True), the implication \(\varphi \to \psi\) evaluates to \(\text{True} \to \text{False}\), which equals False under all assignments, making it unsatisfiable.

  7. if \(\varphi\) is a tautology, then \(\psi \to \varphi\) is a tautology.
    True. Justification: By definition, an implication \(A \to B\) is equivalent to \(\neg A \lor B\). Here, it translates to \(\neg\psi \lor \varphi\). Since \(\varphi\) is a tautology (always True), the disjunction \(\neg\psi \lor \text{True}\) is always True, making the entire expression a tautology.

Exercise 1.8. Variable Accumulation (**)

Verify that:

\(x \land y \equiv x \land y \land (w \lor \neg w)\)

Solution

  1. According to the Law of Excluded Middle, a variable or its negation (\(w \lor \neg w\)) is always True (it is a tautology).
  2. Therefore, substitute \((w \lor \neg w)\) with \(\top\) (True):
    \(x \land y \land (w \lor \neg w) \equiv x \land y \land \top\)
  3. According to the Identity Law of Conjunction, \(A \land \top \equiv A\).
  4. Applying this to our formula:
    \(x \land y \land \top \equiv x \land y\).
    Thus, the equivalence \(x \land y \equiv x \land y \land (w \lor \neg w)\) is fully verified.

1. Describe the NAND operator as a truth table and as a formula.

Formula: \(p \text{ NAND } q \equiv \neg(p \land q)\)

Truth Table:

\(p\) \(q\) \(p \text{ NAND } q\)
001
011

2. Show how one can express conjunction (\(\land\)), disjunction (\(\lor\)), and negation (\(\neg\)) using only NAND.

  • Negation (\(\neg p\)):
    We need an output that is opposite to \(p\).
    \(\neg p \equiv p \text{ NAND } p\)
    (Proof: \(\neg(p \land p) \equiv \neg p\))

  • Conjunction (\(p \land q\)):
    We know \(p \land q\) is the exact inverse of \(p \text{ NAND } q\). Thus, we simply negate a NAND using the negation setup shown above.
    \(p \land q \equiv \neg(p \text{ NAND } q)\)
    \(p \land q \equiv (p \text{ NAND } q) \text{ NAND } (p \text{ NAND } q)\)

  • Disjunction (\(p \lor q\)):
    According to De Morgan's laws, \(p \lor q \equiv \neg(\neg p \land \neg q)\). This matches the definition of a NAND operation on inverted inputs (\((\neg p) \text{ NAND } (\neg q)\)).
    Substitute the negation rule for both \(p\) and \(q\):
    \(p \lor q \equiv (p \text{ NAND } p) \text{ NAND } (q \text{ NAND } q)\)

Exercise 2.1. Knowledge Bases (**)

For the following truth table, construct a knowledge base (expressed as a set of propositional rules) which represents the same knowledge, or argue why it cannot be done.

xyzK
0001
0010
0101
0111
1000
1011
1101
1110

Answer

It cannot be done.

Justification

In the context of knowledge bases, a "propositional rule" restricts us to Horn clauses (rules of the form \(Body \rightarrow Head\), where the body is a conjunction of positive literals and the head is a single positive literal or false).

A fundamental property of any knowledge base composed of Horn clauses is that its set of valid models (satisfying assignments) must be closed under intersection. The intersection of two models is found by performing a bitwise AND on their truth values (i.e., a variable is true in the intersection if and only if it is true in both models).

Let's identify the models from the truth table where \(K = 1\). We represent a model as the set of variables that are assigned 1 (true):

  • \(M_1 = \{\}\) (from 0 0 0)
  • \(M_2 = \{y\}\) (from 0 1 0)
  • \(M_3 = \{y, z\}\) (from 0 1 1)
  • \(M_4 = \{x, z\}\) (from 1 0 1)
  • \(M_5 = \{x, y\}\) (from 1 1 0)

Now, let's take the intersection of models \(M_3\) and \(M_4\):
\(M_3 \cap M_4 = \{y, z\} \cap \{x, z\} = \{z\}\)

This resulting intersection corresponds to the assignment \(x=0, y=0, z=1\). If we look at the truth table for this assignment, the output is \(K = 0\).

Because the intersection of two valid models resulted in an invalid model, the set of assignments is not closed under intersection. Therefore, it is mathematically impossible to represent this exact truth table using a knowledge base of propositional rules (Horn clauses).

Exercise 2.2. Expressivity (**)

Express the following properties as Horn clauses and as propositional rules:

  1. \(x\) must be false
  2. if \(y\) is true, then \(x\) is false
  3. if \(x\) or \(y\) is true then \(z\) is true
  4. if \(x\) is true, then \(y\) and \(z\) are true

Which of these Horn clauses are definite?

Answer

(Note: in rule notation, an implication is usually written as \(Head \leftarrow Body\), and false is used for constraints).

1. \(x\) must be false

  • Horn clause: \(\neg x\)
  • Propositional rule: \(\text{false} \leftarrow x\)

2. if \(y\) is true, then \(x\) is false

  • Horn clause: \(\neg y \vee \neg x\)
  • Propositional rule: \(\text{false} \leftarrow x \wedge y\)

3. if \(x\) or \(y\) is true then \(z\) is true

  • Horn clauses: \((\neg x \vee z)\) and \((\neg y \vee z)\)
  • Propositional rules: \(z \leftarrow x\) and \(z \leftarrow y\)

4. if \(x\) is true, then \(y\) and \(z\) are true

  • Horn clauses: \((\neg x \vee y)\) and \((\neg x \vee z)\)
  • Propositional rules: \(y \leftarrow x\) and \(z \leftarrow x\)

Which of these Horn clauses are definite?

A definite clause is a strict subset of Horn clauses defined as having exactly one positive literal.

  • Clauses for 1 and 2 have zero positive literals (they are constraints).
  • Clauses for 3 and 4 each have exactly one positive literal (the heads \(y\) or \(z\)).

Conclusion: Therefore, the Horn clauses for properties 3 and 4 are definite.

Exercise 2.3. Consistency (**)

Is every knowledge base without constraints consistent? Justify your answer

Answer

Yes, every knowledge base without constraints is consistent.

Justification

A "constraint" is a rule that has false as its head (e.g., \(\text{false} \leftarrow b_1 \wedge \dots \wedge b_n\)). A knowledge base without constraints consists entirely of definite clauses, meaning every single rule has a positive atom as its head.

If you create an interpretation where every variable is assigned true, the head of every rule evaluates to true. Because an implication (\(Body \rightarrow Head\)) is always true when its Head is true, all rules in the knowledge base are satisfied. Since there is at least one valid model (the "all-true" interpretation), the knowledge base is guaranteed to be consistent.

Exercise 2.4. Consistency (***)

Is every knowledge base without facts consistent? Justify your answer

Answer

Yes, every knowledge base without facts is consistent.

Justification

A "fact" is a rule with an empty body (or equivalently, a body that unconditionally evaluates to true, such as \(Head \leftarrow \text{true}\)). In a knowledge base without facts, every rule is guaranteed to have at least one positive atom in its body.

If you create an interpretation where every variable is assigned false, the body of every rule will evaluate to false. An implication with a false antecedent (\(\text{False} \rightarrow Head\)) is vacuously true. Therefore, the "all-false" interpretation will satisfy every rule in the knowledge base without ever triggering a contradiction, meaning the knowledge base is guaranteed to be consistent.

Exercise 2.5. Horn

Question: Build an example that shows why the unit resolution (aka truth propagation) method does not work for clauses which are not Horn.

Solution

A Horn clause is a clause with at most one positive literal. The unit resolution method works by repeatedly finding a unit clause (a clause with only one literal), assigning it the truth value that satisfies it, and propagating that truth value to simplify the remaining clauses. If a set of clauses is Horn and unsatisfiable, unit resolution is guaranteed to find a contradiction (derive the empty clause).

To show that this method is incomplete for non-Horn clauses, we can construct an unsatisfiable knowledge base without any initial unit clauses. Consider the following set of 4 clauses:

  1. \( x \lor y \)
  2. \( x \lor \neg y \)
  3. \( \neg x \lor y \)
  4. \( \neg x \lor \neg y \)

Why unit resolution fails here:

  • Clause (1) is not a Horn clause because it contains two positive literals (\( x \) and \( y \)).
  • The set is logically unsatisfiable because no matter what truth values you assign to \( x \) and \( y \), at least one of these four clauses will evaluate to False.
  • However, if we attempt to apply unit resolution (truth propagation), the method gets stuck immediately. Because there are no unit clauses (clauses with a single literal) available to start the propagation process, the method cannot make any deductions and fails to derive the empty clause (contradiction). This proves that unit resolution is incomplete for non-Horn clauses.

Exercise 2.6. Tableaux

Question: Consider the knowledge base \( K = \{ x \land y, (\neg x \lor z) \land \neg w \} \), which is consistent.

Verify that the valuation \( V \) with \( V(w) = 0, V(x) = V(y) = V(z) = 1 \) satisfies both formulas.

Solution

We evaluate the truth value of each formula in the knowledge base by substituting the given truth values from the valuation \( V \):

  • \( V(w) = 0 \) (False)
  • \( V(x) = 1 \) (True)
  • \( V(y) = 1 \) (True)
  • \( V(z) = 1 \) (True)
  1. First formula: \( F_1 = x \land y \)
    \( V(F_1) = V(x) \land V(y) = 1 \land 1 = 1 \).
    The valuation satisfies \( x \land y \).

  2. Second formula: \( F_2 = (\neg x \lor z) \land \neg w \)
    • First, evaluate \( \neg x \): \( V(\neg x) = \neg 1 = 0 \).
    • Next, evaluate the disjunction \( (\neg x \lor z) \): \( 0 \lor V(z) = 0 \lor 1 = 1 \).
    • Then, evaluate \( \neg w \): \( V(\neg w) = \neg 0 = 1 \).
    • Finally, evaluate the main conjunction: \( V(F_2) = 1 \land 1 = 1 \).
    The valuation satisfies \( (\neg x \lor z) \land \neg w \).

Since \( V \) evaluates both formulas to 1 (True), it is a valid model that satisfies the knowledge base.

Exercise 2.7. Reasoning

Question: Consider the propositional KB \( K \) formed by the following rules:

  • \( w \leftarrow \)
  • \( x \leftarrow \)
  • \( v \leftarrow u, w \)
  • \( y \leftarrow x, z, w \)
  • \( z \leftarrow w, x \)
  • \( \leftarrow y, z \)
  1. Find all the facts that are entailed by \( K \)
  2. Decide which of the following rules are entailed by \( K \):
    • \( u \leftarrow v \)
    • \( z \leftarrow u \)
    • \( t \leftarrow x, v \)

Solution

Let's translate the rules into standard logic and apply forward chaining (truth propagation) to see what \( K \) derives:

  1. \( w \leftarrow \) (Base fact: \( w \) is True)
  2. \( x \leftarrow \) (Base fact: \( x \) is True)
  3. \( v \leftarrow u, w \) (\( u \land w \rightarrow v \))
  4. \( y \leftarrow x, z, w \) (\( x \land z \land w \rightarrow y \))
  5. \( z \leftarrow w, x \) (\( w \land x \rightarrow z \))
  6. \( \leftarrow y, z \) (Constraint: \( y \land z \rightarrow \bot \), meaning \( y \) and \( z \) cannot be both true)

Forward Chaining Derivation:

  • Step 1: From rules 1 and 2, we know \( w \) and \( x \) are True.
  • Step 2: From rule 5 (\( z \leftarrow w, x \)), because \( w \) and \( x \) are True, we derive that \( z \) is True.
  • Step 3: From rule 4 (\( y \leftarrow x, z, w \)), because \( x \), \( z \), and \( w \) are all True, we derive that \( y \) is True.
  • Step 4: From rule 6 (\( \leftarrow y, z \)), because we have derived that both \( y \) and \( z \) are True, this rule triggers a contradiction (\( \bot \)).

Because the knowledge base derives a contradiction, \( K \) is logically inconsistent. In classical logic, according to the Principle of Explosion (ex falso quodlibet), an inconsistent knowledge base entails absolutely everything.

Answers:

  1. Find all the facts that are entailed by \( K \):
    Because \( K \) is inconsistent, every possible fact (propositional variable) in your domain is logically entailed by \( K \) (e.g., \( w, x, y, z, u, v, t \), etc.).
    (Note: If your instructor strictly expects the facts explicitly derived by the forward chaining algorithm before it crashed on the contradiction, those are \( w, x, z \), and \( y \).)

  2. Decide which of the following rules are entailed by \( K \):
    • \( u \leftarrow v \)
    • \( z \leftarrow u \)
    • \( t \leftarrow x, v \)
    Because an inconsistent knowledge base logically entails any arbitrary formula, all of the above rules are entailed by \( K \).

Exercise 2.8. Reasoning (**)

Consider the propositional KB \(K\) formed by the following rules:

\(w \leftarrow\) \(y \leftarrow x, z, w\)
\(x \leftarrow\) \(z \leftarrow w, x, y\)
\(v \leftarrow u, w\) \(y \leftarrow v\)
\(t \leftarrow u, v\) \(\leftarrow y, z\)
  1. Find all the facts that are entailed by \(K\)
  2. Decide which of the following rules are entailed by \(K\):
    • \(s \leftarrow t\)
    • \(u \leftarrow v\)
    • \(z \leftarrow u\)
    • \(t \leftarrow x, v\)
    • \(t \leftarrow s\)

Solution

1. Find all the facts that are entailed by \(K\)

In propositional logic, evaluating a KB means finding its valid models (truth assignments that satisfy all rules). Let's interpret the rules as logical implications:

  1. \(w\) (Fact: \(w\) is True)
  2. \(x\) (Fact: \(x\) is True)
  3. \(u \land w \rightarrow v\)
  4. \(u \land v \rightarrow t\)
  5. \(x \land z \land w \rightarrow y\)
  6. \(w \land x \land y \rightarrow z\)
  7. \(v \rightarrow y\)
  8. \(\neg (y \land z)\) (The rule \(\leftarrow y, z\) is an integrity constraint meaning \(y \land z \rightarrow \text{False}\))

Let's deduce the truth values:

  • From (1) and (2), we know exactly that \(w = \text{True}\) and \(x = \text{True}\).
  • Substituting \(w\) and \(x\) as True into (5) and (6) gives us \(z \rightarrow y\) and \(y \rightarrow z\). This means \(y \leftrightarrow z\) (they must have the exact same truth value).
  • From (8), \(y\) and \(z\) cannot be both True. Because they share the same truth value, they must both be False (\(y = \text{False}, z = \text{False}\)).
  • From (7), \(v \rightarrow y\). Since \(y\) is False, \(v\) must be False.
  • From (3), \(u \land w \rightarrow v\). Since \(w\) is True and \(v\) is False, \(u\) must be False.
  • For \(t\), rule (4) becomes \(\text{False} \rightarrow t\), which is mathematically True regardless of \(t\)'s value. The variable \(s\) is entirely unconstrained.

Therefore, the only positive facts (atoms) strictly entailed by \(K\) are \(w\) and \(x\). (Note: if negative facts are considered, \(\neg y\), \(\neg z\), \(\neg v\), and \(\neg u\) are also entailed).

2. Decide which of the following rules are entailed by \(K\)

A rule \(\text{Head} \leftarrow \text{Body}\) is entailed if \(\text{Body} \rightarrow \text{Head}\) is True in all models of \(K\). Using our established truth values (\(v = \text{False}\) and \(u = \text{False}\)):

  • \(s \leftarrow t \iff (t \rightarrow s)\): Not entailed (e.g., \(t = \text{True}\) and \(s = \text{False}\) is a valid model).
  • \(u \leftarrow v \iff (v \rightarrow u)\): Entailed. Because \(v = \text{False}\), the implication \(\text{False} \rightarrow u\) is always True.
  • \(z \leftarrow u \iff (u \rightarrow z)\): Entailed. Because \(u = \text{False}\), the implication \(\text{False} \rightarrow z\) is always True.
  • \(t \leftarrow x, v \iff (x \land v \rightarrow t)\): Entailed. Because \(v = \text{False}\), the premise \((x \land v)\) is False, making the implication always True.
  • \(t \leftarrow s \iff (s \rightarrow t)\): Not entailed.

Exercise 2.9. KE (*)

Construct a propositional KB expressing the following knowledge:

  • trains, cars, and boats are vehicles
  • land vehicles have wheels
  • boats do not have wheels
  • cars travel on land

Solution

To express this knowledge clearly, we map the concepts to standard propositional variables:

  • \(Train\): "The object is a train"
  • \(Car\): "The object is a car"
  • \(Boat\): "The object is a boat"
  • \(Vehicle\): "The object is a vehicle"
  • \(LandVehicle\): "The object is a land vehicle"
  • \(HasWheels\): "The object has wheels"
  • \(TravelOnLand\): "The object travels on land"

Knowledge Base (KB):

  1. \(Train \rightarrow Vehicle\) (Trains are vehicles)
  2. \(Car \rightarrow Vehicle\) (Cars are vehicles)
  3. \(Boat \rightarrow Vehicle\) (Boats are vehicles)
  4. \(LandVehicle \rightarrow HasWheels\) (Land vehicles have wheels)
  5. \(Boat \rightarrow \neg HasWheels\) (Boats do not have wheels)
  6. \(Car \rightarrow TravelOnLand\) (Cars travel on land)

Implicit Lexical Rule: In order to functionally connect a vehicle that "travels on land" to the compound class "land vehicle", an implicit defining rule is typically required in Knowledge Engineering contexts:

  • 7. \(Vehicle \land TravelOnLand \rightarrow LandVehicle\)

Exercise 2.10. KE II (*)

Using the KB from Exercise 2.9., answer the following questions:

  1. are boats land vehicles?
  2. are trains land vehicles?
  3. do trains have wheels?
  4. do cars have wheels?

Solution

Using classical logical deduction from the KB we just built:

1. Are boats land vehicles?

No. The KB states \(Boat \rightarrow \neg HasWheels\). By contraposition on rule 4 (\(LandVehicle \rightarrow HasWheels\)), we get \(\neg HasWheels \rightarrow \neg LandVehicle\). Thus, we logically deduce \(Boat \rightarrow \neg LandVehicle\).

2. Are trains land vehicles?

Unknown (or No, assuming a Closed-World Assumption). In classical logic, the KB only states \(Train \rightarrow Vehicle\). We have no premises connecting trains to traveling on land, so we cannot logically deduce whether they are land vehicles or not.

3. Do trains have wheels?

Unknown (or No, assuming a Closed-World Assumption). For the same reason as above, without knowing if they are land vehicles, there is no rule path to logically deduce \(HasWheels\).

4. Do cars have wheels?

Yes. We know \(Car \rightarrow Vehicle\) (Rule 2) and \(Car \rightarrow TravelOnLand\) (Rule 6). Applying our bridging rule (Rule 7), this combination proves they are land vehicles (\(Car \rightarrow LandVehicle\)). Because all land vehicles have wheels (Rule 4), we deduce logically that \(Car \rightarrow HasWheels\).

3 Predicate Knowledge Bases

Exercise 3.1. KE (*)

Build a small predicate KB that describes the notions of (at least) Teacher, Student, Course, teaches, and attends in the context of a university

Solution

To build a predicate Knowledge Base (KB), we first define the vocabulary (predicates) and then write First-Order Logic (FOL) axioms to describe the relationships between these concepts.

Predicates:

  • Unary predicates: \(Teacher(x)\), \(Student(x)\), \(Course(x)\)
  • Binary predicates: \(teaches(x, y)\), \(attends(x, y)\)

Knowledge Base (Axioms):

  1. Only teachers teach courses, and what they teach are courses:
    \(\forall x \forall y \ (teaches(x, y) \implies Teacher(x) \land Course(y))\)
  2. Only students attend courses, and what they attend are courses:
    \(\forall x \forall y \ (attends(x, y) \implies Student(x) \land Course(y))\)
  3. Teachers and students are distinct roles (disjoint):
    \(\forall x \ (Teacher(x) \implies \neg Student(x))\)
  4. Every student attends at least one course:
    \(\forall x \ (Student(x) \implies \exists y \ (Course(y) \land attends(x, y)))\)
  5. Every course is taught by at least one teacher:
    \(\forall x \ (Course(x) \implies \exists y \ (Teacher(y) \land teaches(y, x)))\)

Exercise 3.2. KE (**)

Construct a predicate KB expressing the following knowledge:

  • echidnas and platypi are monotremata
  • echidnas are not venomous
  • platypi never attack each other
  • Plato attacks Echo
  • Plato and Echo are monotremata

Can you deduce whether Plato or Echo are echidnas or platypi?

Solution

First, let's translate the given English sentences into a formal predicate KB.
Let \(E(x)\) = echidna, \(P(x)\) = platypus, \(M(x)\) = monotreme, \(V(x)\) = venomous, and \(A(x, y)\) = x attacks y.

Knowledge Base:

  1. Echidnas and platypi are monotremata:
    \(\forall x \ (E(x) \implies M(x))\)
    \(\forall x \ (P(x) \implies M(x))\)
  2. Echidnas are not venomous:
    \(\forall x \ (E(x) \implies \neg V(x))\)
  3. Platypi never attack each other:
    \(\forall x \forall y \ (P(x) \land P(y) \implies \neg A(x, y))\)
  4. Plato attacks Echo:
    \(A(\text{Plato}, \text{Echo})\)
  5. Plato and Echo are monotremata:
    \(M(\text{Plato}) \land M(\text{Echo})\)

Deduction:

Can you deduce whether Plato or Echo are echidnas or platypi?

No, we cannot deduce their exact species. However, we can deduce one specific constraint: they cannot both be platypi.

Proof: Axiom 3 states that if two animals are both platypi, they do not attack each other. Because we know as a fact that Plato attacks Echo (\(A(\text{Plato}, \text{Echo})\) is True), by Modus Tollens, the premise that they are both platypi must be False: \(\neg (P(\text{Plato}) \land P(\text{Echo}))\).

They could be a mix (one platypus, one echidna), both echidnas, or even another undiscovered type of monotreme.

Exercise 3.3. KE II (**)

Considering the KB from Exercise 3.2.,

  1. what information do you need to be able to deduce that Echo is not an echidna?
  2. and to know that Echo is a platypus?

Solution

Building on the KB from Exercise 3.2:

1. What information do you need to be able to deduce that Echo is not an echidna?

You would need the information that Echo is venomous (\(V(\text{Echo})\)).

Reasoning: We know from the KB that echidnas are not venomous (\(\forall x \ (E(x) \implies \neg V(x))\)). By the logical rule of contraposition, anything that is venomous cannot be an echidna (\(\forall x \ (V(x) \implies \neg E(x))\)). Therefore, if we learn \(V(\text{Echo})\), we can deduce \(\neg E(\text{Echo})\).

2. And to know that Echo is a platypus?

In addition to the deduction above (\(\neg E(\text{Echo})\)), you would need the information that all monotremata are either echidnas or platypi (i.e., there are no other types of monotremata).

Reasoning: The KB states that Plato and Echo are monotremata (\(M(\text{Echo})\)), but it currently lacks a rule restricting monotremata strictly to those two animals. If we add the rule \(\forall x \ (M(x) \implies E(x) \lor P(x))\), we can use the Disjunctive Syllogism. We know Echo is a monotreme, so she must be either an echidna or a platypus. Since we deduced in step 1 that she is not an echidna, it definitively leaves us with the deduction that Echo is a platypus (\(P(\text{Echo})\)).

Exercise 3.4. Grounding (*)

  1. Construct the grounding of the KB:

    \(\mathcal{K} = \{ P(a) \leftarrow , \quad R(b) \leftarrow , \quad Q(b, a) \leftarrow , \)
    \(\quad\quad\quad P(x) \leftarrow Q(x, a), \)
    \(\quad\quad\quad Q(c, x) \leftarrow P(x) \}\)

  2. Use this grounding to verify whether the following rules are entailed by \(\mathcal{K}\):

    1. \(P(c)\)
    2. \(Q(a, b)\)
    3. \(Q(c, c)\)
    4. \(R(c)\)
    5. \(R(c) \leftarrow Q(a, c)\)
    6. \(Q(c, a) \leftarrow Q(a, c)\)

Solution

1. Construct the grounding of the KB:

First, we identify the set of constants appearing in the knowledge base (\(\mathcal{K}\)), which are \(a, b,\) and \(c\). Grounding the KB means substituting all variables (in this case, just \(x\)) in the rules with every possible constant from the domain.

The original KB contains 3 facts (which are already grounded) and 2 rules with variables. Grounding produces the following:

Facts:

  • \(P(a) \leftarrow\)
  • \(R(b) \leftarrow\)
  • \(Q(b, a) \leftarrow\)

Grounding \(P(x) \leftarrow Q(x, a)\) for \(x \in \{a, b, c\}\):

  • \(P(a) \leftarrow Q(a, a)\)
  • \(P(b) \leftarrow Q(b, a)\)
  • \(P(c) \leftarrow Q(c, a)\)

Grounding \(Q(c, x) \leftarrow P(x)\) for \(x \in \{a, b, c\}\):

  • \(Q(c, a) \leftarrow P(a)\)
  • \(Q(c, b) \leftarrow P(b)\)
  • \(Q(c, c) \leftarrow P(c)\)

2. Use this grounding to verify whether the following rules are entailed by \(\mathcal{K}\):

To verify entailment, we can compute the minimal model (Least Herbrand Model) of \(\mathcal{K}\) by applying the grounded rules starting from the initial facts to see all the facts that are guaranteed to be true:

  1. Initial facts: \(\{P(a), R(b), Q(b, a)\}\)
  2. Applying \(P(b) \leftarrow Q(b, a)\) adds \(P(b)\)
  3. Applying \(Q(c, a) \leftarrow P(a)\) adds \(Q(c, a)\)
  4. Applying \(P(c) \leftarrow Q(c, a)\) adds \(P(c)\)
  5. Applying \(Q(c, b) \leftarrow P(b)\) adds \(Q(c, b)\)
  6. Applying \(Q(c, c) \leftarrow P(c)\) adds \(Q(c, c)\)

The minimal model (\(\mathcal{M}\)) is: \(\{P(a), R(b), Q(b, a), P(b), Q(c, a), P(c), Q(c, b), Q(c, c)\}\). A fact is entailed if it is contained in this minimal model.

  • a) \(P(c)\): Yes, it is entailed. (\(P(c)\) is derived in step 4 and is in \(\mathcal{M}\)).
  • b) \(Q(a, b)\): No, it is not entailed. (\(Q(a, b)\) is not in \(\mathcal{M}\)).
  • c) \(Q(c, c)\): Yes, it is entailed. (\(Q(c, c)\) is derived in step 6 and is in \(\mathcal{M}\)).
  • d) \(R(c)\): No, it is not entailed. (\(R(c)\) is not in \(\mathcal{M}\)).
  • e) \(R(c) \leftarrow Q(a, c)\): No, it is not entailed.
    To check if an implication rule \(H \leftarrow B\) is entailed, we add the premise \(B\) to our facts and see if \(H\) can be derived. If we add \(Q(a, c)\) to our minimal model \(\mathcal{M}\), no new rules are triggered because \(Q(a, c)\) does not match the body of any available rule (e.g., \(P(x) \leftarrow Q(x, a)\) specifically requires the second argument to be \(a\)). Since \(R(c)\) is still not derived, there is a valid model where \(Q(a, c)\) is true but \(R(c)\) is false.
  • f) \(Q(c, a) \leftarrow Q(a, c)\): Yes, it is entailed.
    \(Q(c, a)\) is already a proven fact in the minimal model \(\mathcal{M}\) (derived in step 3), meaning it is unconditionally true in every model of \(\mathcal{K}\). Therefore, the implication "\(Q(c, a)\) is true IF \(Q(a, c)\) is true" is trivially satisfied in all models.

Exercise 3.5. Grounding (**)

Consider a predicate KB that uses 5 different constants, and contains the rule

\(Q(x, y) \leftarrow R(x), Q(x, y), S(y, z), Q(z, x)\)

How many rules will its grounding produce? Explain how you got this number.

Solution

Total generated rules: 125 rules.

Explanation:

To construct a full grounding of a rule, you must substitute every distinct variable in the rule with every possible combination of constants from the domain.

  • The given rule contains 3 distinct variables: \(x\), \(y\), and \(z\).
  • The problem states that the knowledge base uses 5 different constants.

Because the substitution choice for each variable is independent of the others, you use the multiplication principle to find the total combinations.

Total ground rules = (Choices for \(x\)) \(\times\) (Choices for \(y\)) \(\times\) (Choices for \(z\))
Total ground rules = \(5 \times 5 \times 5 = 5^3 = \mathbf{125}\).

Exercise 4.1. Satisfiability

Check which of the following \(\mathcal{EL}^\perp\) concepts are satisfiable and which are not:

  • \(\exists r.(\exists s.A \sqcap B) \sqcap \exists s.\top\)
  • \(A \sqcap C\)
  • \(\exists r.\perp \sqcap B\)
  • \(\exists r.\exists r.\exists s.A\)
  • \(\exists r.\exists r.\exists s.\perp\)

Solution

In \(\mathcal{EL}^\perp\), a concept is unsatisfiable if and only if it implies the bottom concept (\(\perp\)). Since there is no negation in \(\mathcal{EL}^\perp\), simply checking for the unavoidable presence of \(\perp\) is sufficient.

  • \(\exists r.(\exists s.A \sqcap B) \sqcap \exists s.\top\): Satisfiable
    (No \(\perp\) is present or implied).
  • \(A \sqcap C\): Satisfiable
    (Standard conjunction of concept names; no contradictions).
  • \(\exists r.\perp \sqcap B\): Unsatisfiable
    (The term \(\exists r.\perp\) requires an \(r\)-successor to belong to the empty set \(\perp\), which is impossible. Thus \(\exists r.\perp \equiv \perp\), making the whole expression \(\perp\)).
  • \(\exists r.\exists r.\exists s.A\): Satisfiable
    (A simple chain of roles leading to \(A\); no \(\perp\) is present).
  • \(\exists r.\exists r.\exists s.\perp\): Unsatisfiable
    (\(\exists s.\perp \equiv \perp\), which propagates outwards making \(\exists r.\perp \equiv \perp\), and again making the entire concept equivalent to \(\perp\)).

Exercise 4.2. Satisfiability II

For each of the satisfiable concepts in Exercise 4.1., find an interpretation that makes it non-empty.

Solution

We need to provide an interpretation \(\mathcal{I} = (\Delta^\mathcal{I}, \cdot^\mathcal{I})\) for each of the three satisfiable concepts from 4.1 such that the concept interpretation is non-empty (i.e., \(C^\mathcal{I} \neq \emptyset\)).

1. \(C_1 = \exists r.(\exists s.A \sqcap B) \sqcap \exists s.\top\)

  • Domain: \(\Delta^\mathcal{I} = \{1, 2, 3, 4\}\)
  • Roles: \(r^\mathcal{I} = \{(1, 2)\}\),   \(s^\mathcal{I} = \{(1, 3), (2, 4)\}\)
  • Concepts: \(A^\mathcal{I} = \{4\}\),   \(B^\mathcal{I} = \{2\}\)
  • Verification: Element \(2\) is in \(B\) and has an \(s\)-successor (\(4\)) in \(A\), so \(2 \in (\exists s.A \sqcap B)^\mathcal{I}\). Element \(1\) has an \(r\)-successor (\(2\)) in this set, so \(1 \in (\exists r.(\exists s.A \sqcap B))^\mathcal{I}\). Element \(1\) also has an \(s\)-successor (\(3\)) in \(\top\), so \(1 \in C_1^\mathcal{I}\). The concept is non-empty.

2. \(C_2 = A \sqcap C\)

  • Domain: \(\Delta^\mathcal{I} = \{1\}\)
  • Roles: \(\emptyset\)
  • Concepts: \(A^\mathcal{I} = \{1\}\),   \(C^\mathcal{I} = \{1\}\)
  • Verification: Element \(1 \in A^\mathcal{I}\) and \(1 \in C^\mathcal{I}\), thus \(1 \in (A \sqcap C)^\mathcal{I}\). The concept is non-empty.

3. \(C_3 = \exists r.\exists r.\exists s.A\)

  • Domain: \(\Delta^\mathcal{I} = \{1, 2, 3, 4\}\)
  • Roles: \(r^\mathcal{I} = \{(1, 2), (2, 3)\}\),   \(s^\mathcal{I} = \{(3, 4)\}\)
  • Concepts: \(A^\mathcal{I} = \{4\}\)
  • Verification: Element \(3\) has an \(s\)-successor in \(A\) (\(4\)). Element \(2\) has an \(r\)-successor in \(\exists s.A\) (\(3\)). Element \(1\) has an \(r\)-successor in \(\exists r.\exists s.A\) (\(2\)). Thus \(1 \in C_3^\mathcal{I}\). The concept is non-empty.

Exercise 4.3. Subsumption

Check whether the following subsumptions hold:

  • \(\exists r.(A \sqcap B) \sqsubseteq \exists r.A \sqcap \exists s.B\)
  • \(A \sqcap B \sqcap \exists s.A \sqsubseteq \exists s.(B \sqcap A)\)
  • \(\exists s.(A \sqcap B) \sqsubseteq \exists s.A \sqcap \exists s.B\)
  • \(\exists r.(A \sqcap \perp) \sqsubseteq \exists r.A \sqcap \exists s.B\)
  • \(\exists r.(\exists s.A \sqcap B) \sqsubseteq \exists r.B \sqcap \exists r.\exists s.A\)
  • \(\exists r.(A \sqcap \exists r.(B \sqcap \exists s.A)) \sqsubseteq \exists r.A \sqcap \exists r.\exists r.B\)
  • \(A \sqcap B \sqcap \exists s.A \sqsubseteq \exists s.(B \sqcap \perp)\)

Solution

We evaluate whether \(C \sqsubseteq D\) holds (meaning \(C^\mathcal{I} \subseteq D^\mathcal{I}\) for all interpretations \(\mathcal{I}\)).

  • \(\exists r.(A \sqcap B) \sqsubseteq \exists r.A \sqcap \exists s.B\) → Does NOT hold (False)
    Reason: The left side guarantees an \(r\)-successor that is in both \(A\) and \(B\). The right side demands an \(r\)-successor in \(A\) and an \(s\)-successor in \(B\). The left side provides no guarantee that an \(s\)-successor exists at all.
  • \(A \sqcap B \sqcap \exists s.A \sqsubseteq \exists s.(B \sqcap A)\) → Does NOT hold (False)
    Reason: The left side says the current node is in \(B\), and it has an \(s\)-successor in \(A\). The right side requires an \(s\)-successor that is in both \(B\) and \(A\). The \(s\)-successor is only guaranteed to be in \(A\), not necessarily in \(B\).
  • \(\exists s.(A \sqcap B) \sqsubseteq \exists s.A \sqcap \exists s.B\) → Holds (True)
    Reason: If a node has an \(s\)-successor that is in both \(A\) and \(B\), then that exact same successor serves as an \(s\)-successor in \(A\) and an \(s\)-successor in \(B\).
  • \(\exists r.(A \sqcap \perp) \sqsubseteq \exists r.A \sqcap \exists s.B\) → Holds (True)
    Reason: The concept \(A \sqcap \perp\) is equivalent to \(\perp\). Thus, the left-hand side is \(\exists r.\perp \equiv \perp\). Because the bottom concept (\(\perp\)) is empty in every interpretation, it is a subset of every concept. Ex Falso Quodlibet.
  • \(\exists r.(\exists s.A \sqcap B) \sqsubseteq \exists r.B \sqcap \exists r.\exists s.A\) → Holds (True)
    Reason: If a node has an \(r\)-successor that belongs to both \(B\) and \(\exists s.A\), then that single successor fulfills the condition for having an \(r\)-successor in \(B\) and an \(r\)-successor in \(\exists s.A\).
  • \(\exists r.(A \sqcap \exists r.(B \sqcap \exists s.A)) \sqsubseteq \exists r.A \sqcap \exists r.\exists r.B\) → Holds (True)
    Reason: The left side indicates the node has an \(r\)-successor (let's call it \(y\)) such that \(y \in A\) and \(y \in \exists r.(B \sqcap \exists s.A)\). Because \(y \in A\), the original node is in \(\exists r.A\). Because \(y\) has an \(r\)-successor (let's call it \(z\)) in \(B\), \(y \in \exists r.B\), which makes the original node an element of \(\exists r.\exists r.B\). Both conditions of the right side are met.
  • \(A \sqcap B \sqcap \exists s.A \sqsubseteq \exists s.(B \sqcap \perp)\) → Does NOT hold (False)
    Reason: The right side contains \(B \sqcap \perp \equiv \perp\), so the entire right side evaluates to \(\exists s.\perp \equiv \perp\). The left side does not contain any contradictions and is satisfiable (can be non-empty), so it cannot be subsumed by the empty set (\(\perp\)).
